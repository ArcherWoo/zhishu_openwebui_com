## 五、PowerPoint 离线解析检查

### 1. `.pptx`

`.pptx` 主要依赖 `MarkItDown`。

需要确认：

- [ ] `ppt_offline_runtime/wheels/` 内容完整
- [ ] 项目启动安装依赖时已能从本地 wheel 找到相关包
- [ ] `.pptx` 不依赖 `LibreOffice` 也能解析

### 2. `.ppt`

`.ppt` 除了 `MarkItDown`，还额外依赖 `LibreOffice/soffice`。

需要确认：

- [ ] 已安装 LibreOffice
- [ ] `soffice` 命令可用
- [ ] `.ppt -> .pptx` 转换可执行

在服务器上可手工验证：

```powershell
soffice --version
```

如果这条命令不通：

- `.pptx` 可能仍可用
- `.ppt` 一定会失败

## 十、常见问题排查清单

### 问题 1：上传后前端一直转圈

按顺序检查：

- [ ] 解密后的文件是否真的生成
- [ ] 解密后的扩展名是否正确
- [ ] 后台是否进入对应 loader
- [ ] `.pptx` 是否已经调起 `MarkItDown`
- [ ] `.ppt` 是否已经调起 `soffice`

### 问题 2：`.pptx` 不行，但 Excel 正常

优先检查：

- [ ] `markitdown` 相关 wheel 是否完整
- [ ] Python 依赖是否已从本地离线 wheel 安装

### 问题 3：`.ppt` 不行，但 `.pptx` 正常

优先检查：

- [ ] LibreOffice 是否已安装
- [ ] `soffice --version` 是否能执行
- [ ] `PPT_CONVERTER_COMMAND` 是否正确

### 问题 4：文件能入库，但问答质量差

优先检查：

- [ ] `embedding_model/` 是否为空
- [ ] sentence-transformers 本地模型是否可用
- [ ] embedding 配置是否真的指向本地模型

### 问题 5：启动时报 NLTK data 缺失

优先检查：

- [ ] `nltk_data/` 是否在项目根目录
- [ ] `NLTK_DATA` 是否指向正确目录
- [ ] `AUTO_DOWNLOAD_NLTK=False` 是否生效

### 问题 6：启动时报 pyodide 缺失

优先检查：

- [ ] `vendor/npm/` 是否完整
- [ ] `static/pyodide/` 是否完整
- [ ] 前端相关离线资源是否已恢复
