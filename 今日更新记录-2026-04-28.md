# 今日更新记录 - 2026-04-28

## 一句话总结

今天围绕 Open WebUI 内网部署版本做了三类优化：默认中文体验、设置页品牌清理、知识库 Markdown 文档编辑体验升级，并补充了对应的自动化测试和验证记录。

## 主要工作

### 1. 默认中文体验优化

- 将首次进入对话页的默认建议问题改为中文内容，避免内网用户看到英文示例。
- 新增默认中文系统提示词，要求模型优先使用简体中文回答，降低建议、追问、思考辅助内容偶发英文的概率。
- 调整 OpenAI 与 Ollama 请求参数合并逻辑，让默认模型参数能稳定注入到不同后端请求路径。
- 优化 follow-up 追问生成提示词，要求追问默认输出简体中文。
- 新增 `tests/test_chinese_defaults.py`，覆盖默认中文建议、默认中文系统提示、追问提示词和请求参数合并逻辑。

### 2. 设置页“关于”信息清理

- 清理“设置 -> 关于”页面里不适合内网部署展示的官方社区与外链内容。
- 移除 Discord、Open WebUI、X、GitHub star、Twemoji 授权说明、版权和作者等信息。
- 保留产品名、当前版本和 Ollama 版本信息，页面更符合内网产品交付场景。
- 新增 `tests/test_about_settings_pruned.py`，防止这些外链和版权文案后续被误恢复。

### 3. 知识库 Markdown 文档编辑体验升级

- 重做知识库 Markdown 文档工作台的视觉密度，扩大文档阅读宽度，减少左右留白和过重阴影。
- 修复点击内容无法进入编辑的问题：改用 `event.composedPath()` 判断点击来源，避免节点重渲染后误判成“点击外部”。
- 文本块进入编辑后自动聚焦，按 `Esc` 或点击空白处可退出编辑。
- 表格进入编辑后自动聚焦第一个可编辑格子。
- 表格支持编辑表头、新增列、删除列、新增行、复制行、删除行，并继续回写为标准 Markdown 表格。
- 将表格操作按钮从大段文字按钮改为轻量图标按钮，降低视觉噪声。
- 将表格工具条升级为右上角半透明吸顶胶囊，新增行/新增列入口更轻、更像文档编辑器。
- 增加表格行列 hover/focus 弱高亮，鼠标或键盘聚焦某个单元格时，同步提示当前行和当前列。
- 新增 `knowledgeMarkdownTableEditorState.ts` 和对应测试，锁定表格行列高亮状态逻辑。

## 技术实现要点

- 后端默认参数通过 `merge_model_params` 合并，避免模型自定义参数覆盖默认中文系统提示时出现丢失。
- Markdown 表格解析与序列化继续保持轻量实现，不引入 Tiptap 全量富文本重构，降低改动风险。
- 表格数据操作集中在 `knowledgeMarkdownTables.ts`，UI 组件只负责交互呈现和事件回传。
- 编辑器点击外部退出逻辑拆到 `knowledgeMarkdownEditorState.ts`，并用单元测试覆盖“点击后 DOM 重渲染”的边界情况。
- 表格 hover 状态拆到 `knowledgeMarkdownTableEditorState.ts`，避免模板里堆复杂判断。
